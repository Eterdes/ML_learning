# LLM Customer Support Pipeline (Mini-Product)

Продуктовый конвейер на базе локальной языковой модели **Qwen 2.5 (3B)** для автоматического анализа, классификации и генерации ответов на обращения клиентов.

## Что умеет проект
1. **Классифицирует обращения** по 5 категориям (`support`, `sales`, `complaint`, `feedback`, `general_question`).
2. **Определяет тональность** (`positive`, `neutral`, `negative`).
3. **Генерирует лаконичное саммари** обращения (до 200 символов).
4. **Маршрутизирует запрос** на нужного виртуального специалиста и формирует ответ по строгому бизнес-регламенту.
5. **Проверяет себя на адекватность** (Self-Check + Рефлексия) перед отправкой ответа.
6. **Отказоустойчив:** использует ретраи при сбоях сети, обрабатывает кривой JSON от модели и переходит в fallback-режим при полном отказе LLM.


### Ключевые компоненты:
1. **Слой извлечения метаданных (Step 1):** Модель классифицирует текст (одна из 5 категорий), определяет тональность (positive/neutral/negative) и генерирует краткую выжимку (до 100 символов). Результат валидируется схемой **Pydantic**.
2. **Динамический роутинг (Step 2):** На основе извлеченной категории Python выбирает один из 5 специализированных системных промптов (для техподдержки, жалоб, продаж и т.д.).
3. **Контур самопроверки и рефлексии (Step 3-4):** Модель в роли инспектора проверяет финальный ответ на вежливость и соответствие контексту. Если обнаружен брак (например, галлюцинации или некорректный тон), ответ отправляется на доработку с указанием конкретных замечаний (до 2-х итераций).
4. **Слой отказоустойчивости (Guardrails):** - **Retry Policy:** Повторные попытки при временной недоступности Ollama API или тайм-аутах.
   - **Graceful Fallback:** Если модель вернула невалидный JSON, отсутствуют обязательные поля или API полностью недоступно, система автоматически подменяет метаданные и генерирует безопасный дефолтный ответ-заглушку. Пайплайн никогда не падает с критической ошибкой.

## Как установить и запустить

### 1. Подготовка окружения
Убедись, что у тебя установлен Python 3.10+ и запущен локальный сервер **Ollama** с моделью `qwen2.5:3b` (или `qwen2.5:latest`).

Проверить запуск Ollama локально можно командой:
```bash
ollama run qwen2.5:3b "Привет, ты тут?"
```

### 2. Установка зависимостей
Клонируй проект, перейди в папку и установи библиотеки:
```
pip install -r requirements.txt
```

### 3. Режим 1: Прогон демо-сценариев (CLI)
Запусти скрипт напрямую, чтобы прогнать 5 заложенных тестовых кейсов из demo_inputs.json и сохранить лог работы:
```
python main.py
```
Результаты разбора сохранятся в файл demo_results.json, а подробный системный лог — в pipeline.log.

### 4. Режим 2: Запуск веб-сервера (FastAPI API)
Чтобы поднять сервер и интегрировать пайплайн в веб-приложение, запусти:
```
uvicorn main:app --reload --port 8000
```
После этого API будет доступно по адресу: http://localhost:8000
Интерактивная документация Swagger (где можно потыкать эндпоинты ручками) откроется по адресу: http://localhost:8000/docs

Пример curl-запроса к API:
```
curl -X POST "http://localhost:8000/analyze" \
     -H "Content-Type: application/json" \
     -d '{"text": "Не могу оплатить заказ картой, выдает ошибку сети!"}'
```
